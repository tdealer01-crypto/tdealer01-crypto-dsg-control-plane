# DSG Safe DOM Mirror Executor

## Purpose

DSG Safe DOM Mirror Executor is a minimal agent execution design for browser-based work. The agent never receives the full page, raw selectors, tokens, browser cookies, file-system access, or API keys. DSG builds a filtered DOM view, exposes only safe capabilities, and lets the agent choose from what remains.

Core rule:

```text
Agent cannot click what DSG does not expose.
Executor cannot execute what is not in the safe manifest.
```

This design intentionally avoids starting with MCP, CLI plugins, broad API gateways, provider-specific protocol layers, or direct browser automation permissions. Those can be added later only if they prove necessary.

## User benefit

- The agent can work freely inside a safe action world.
- Dangerous controls are removed before the agent sees them.
- Secret values are redacted before observation.
- The executor only accepts element IDs generated by DSG.
- Every action has before/after evidence.

## High-level flow

```text
Real browser / VM / web app
  -> DOM extractor
  -> DSG filter
  -> safe DOM view
  -> agent selects exposed elementId
  -> DSG execute gate
  -> executor clicks/types/scrolls
  -> observe after
  -> evidence log
```

## Safe DOM view

The agent receives a small JSON view, not the full DOM.

```json
{
  "frameId": "frame_001",
  "url": "https://github.com/example/repo/pull/698",
  "title": "Pull Request #698",
  "elements": [
    {
      "id": "e001",
      "role": "link",
      "text": "Files changed",
      "allowedOps": ["click"]
    },
    {
      "id": "e002",
      "role": "button",
      "text": "Show all checks",
      "allowedOps": ["click"]
    },
    {
      "id": "e003",
      "role": "textbox",
      "label": "Search",
      "allowedOps": ["type"]
    }
  ]
}
```

The agent must not receive raw selectors. It only sees stable element IDs for the current frame.

## Server-side manifest

The executor stores the real selector and risk metadata server-side.

```ts
type SafeElementManifest = {
  frameId: string;
  elementId: string;
  selectorHash: string;
  internalSelector: string;
  role: string;
  text: string;
  allowedOps: Array<'click' | 'type' | 'scroll' | 'press'>;
  risk: 'LOW';
  expiresAt: string;
};
```

The manifest is the enforcement boundary. If an agent guesses an element ID or raw selector, execution is blocked.

## Command model

```ts
type SafeDomCommand =
  | { frameId: string; op: 'click'; elementId: string }
  | { frameId: string; op: 'type'; elementId: string; text: string }
  | { frameId: string; op: 'scroll'; elementId: string; direction: 'up' | 'down' }
  | { frameId: string; op: 'press'; key: 'Enter' | 'Escape' | 'Tab' | 'Backspace' };
```

No coordinate commands, JavaScript execution, direct selectors, direct network calls, raw token use, or file access are allowed in the MVP.

## Execute gate

```ts
function verifyCommand(command: SafeDomCommand, manifest: SafeElementManifest[]) {
  const element = manifest.find(
    (item) =>
      item.frameId === command.frameId &&
      item.elementId === command.elementId,
  );

  if (!element) {
    return { decision: 'BLOCK', reason: 'ELEMENT_NOT_EXPOSED_TO_AGENT' };
  }

  if (!element.allowedOps.includes(command.op)) {
    return { decision: 'BLOCK', reason: 'OP_NOT_ALLOWED_FOR_ELEMENT' };
  }

  if (new Date(element.expiresAt).getTime() < Date.now()) {
    return { decision: 'BLOCK', reason: 'SAFE_VIEW_EXPIRED' };
  }

  return { decision: 'ALLOW', element };
}
```

## Filtering policy

Use a simple rule first:

```text
Forbidden = remove from agent view
Sensitive = redact
Risky but sometimes needed = one-time capability after human approval
Safe = expose
```

Default removed controls:

```text
delete
remove
destroy
confirm
pay
purchase
billing
deploy production
merge
publish
post
send
invite
rotate secret
change permission
export private data
```

Default redacted values:

```text
password
token
api key
secret
private key
session cookie
customer data
email / phone where not task-required
long hidden values
```

## One-time dangerous capability

Dangerous controls are not exposed by default. If a destructive action is genuinely required:

```text
agent explains intent
-> DSG pauses
-> human approves
-> DSG exposes exactly one dangerous element
-> short TTL, for example 30 seconds
-> typed confirmation phrase required where possible
-> capability is revoked after one use
-> before/after evidence is recorded
```

Example:

```json
{
  "capabilityId": "cap_delete_001",
  "elementId": "danger_001",
  "op": "click",
  "resource": "deployment dpl_123",
  "approvedBy": "owner",
  "expiresInMs": 30000,
  "requiresTypedPhrase": "DELETE dpl_123"
}
```

## Evidence record

```ts
type DomMirrorEvidence = {
  evidenceId: string;
  sessionId: string;
  frameBeforeId: string;
  command: SafeDomCommand;
  decision: 'ALLOW' | 'BLOCK' | 'REVIEW';
  reason?: string;
  elementText?: string;
  elementRole?: string;
  frameAfterId?: string;
  screenshotBeforeId?: string;
  screenshotAfterId?: string;
  timestamp: string;
};
```

## Non-goals for the first PR

- No production browser automation implementation.
- No direct Facebook, Stripe, GitHub, or banking automation.
- No bypassing provider terms or platform permissions.
- No raw credential access to agents.
- No claim that the system is production-ready from design docs alone.

## Acceptance principle

The MVP is acceptable only when the executor enforces this invariant:

```text
If an element was not exposed in the current unexpired safe manifest, the agent cannot invoke it.
```
