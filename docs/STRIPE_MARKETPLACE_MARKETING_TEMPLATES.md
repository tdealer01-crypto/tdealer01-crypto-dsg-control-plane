## Section 6: Objection Handling

### Objection 1: "We already have fraud prevention (Stripe Radar)"

**Your response:**
"Stripe Radar is excellent for flagging risky transactions. DSG ONE 
is complementary—it governs what happens next. You set policies 
(amount limits, velocity checks, customer rules), and DSG ONE enforces 
them automatically.

Think of it as:
- Stripe Radar: Identifies risk
- DSG ONE: Enforces your response (block, review, allow)

Many customers use both together."

---

### Objection 2: "This adds latency to our payments"

**Your response:**
"Policy evaluation is < 500ms (p99). Most payment processing adds 
2-3 seconds of latency anyway. And if a transaction is blocked, 
that's better than processing fraud and dealing with chargebacks 
45 days later.

You can also use async mode if you need true zero-latency—record 
the decision after the payment posts, then refund if needed."

---

## Section 7: FAQ Templates

### 7.1 General FAQ

**Q: How long does setup take?**
A: 5 minutes. One-click OAuth integration with Stripe. Your first 
policy is live in under 30 minutes.

**Q: Do I need to change my code?**
A: No. Policies are evaluated via Stripe webhooks and our API. If 
using webhooks, zero code changes needed.

**Q: What happens if DSG ONE goes down?**
A: Operations continue to post (we don't block by default). We have 
99.5% uptime SLA. Incidents are rare and trigger auto-failover.

**Q: Can I use this for test mode?**
A: Yes. Separate test mode keys. Test policies without affecting 
live operations.

**Q: How much data retention do you provide?**
A: Audit trails kept 7 years (standard for compliance). Configurable 
per contract.

**Q: Is my data encrypted?**
A: Yes. TLS 1.3 in transit, AES-256 at rest (Supabase managed 
encryption). API keys stored encrypted.

**Q: Can I export my audit trail?**
A: Yes. JSON, CSV, or PDF formats. Audit-ready format for regulators.

### 7.2 Technical FAQ

**Q: What Stripe events are supported?**
A: charge.created, payment_intent.*, payout.created, refund.created, 
and more. Full list in API docs.

**Q: Can I use this with Stripe Connect?**
A: Yes. Per-account policies supported. Each Stripe account gets its 
own policy set.

**Q: What's the API rate limit?**
A: Free: 100 req/min. Professional: 10K req/min. Enterprise: Custom.

**Q: Do you support webhooks?**
A: Yes. Stripe webhooks are the primary integration method. We 
also support polling for advanced use cases.

**Q: Can I automate policy changes?**
A: Yes. Policy update API available. Deploy policy changes
