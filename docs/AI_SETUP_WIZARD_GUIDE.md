# DSG AI Setup Wizard Guide

Complete guide to setting up OpenRouter AI models for DSG ONE.

## Overview

The AI Setup Wizard provides two paths to configure your AI models:

1. **Web UI Wizard** — Interactive 5-step form in the browser
2. **CLI Interactive Wizard** — Command-line setup tool

Both paths generate the same configuration and support:
- 5 free OpenRouter models
- Primary model selection
- Fallback chain configuration
- Usage tracking and billing integration

## Quick Start

### Option A: Web UI (Recommended)

1. Open `https://your-app/setup/ai` in your browser
2. Follow the 5 steps
3. Download or copy your configuration
4. Apply to your environment

### Option B: CLI

```bash
node scripts/setup-wizard.js
```

The CLI will:
1. Prompt for your OpenRouter API key
2. Guide model selection
3. Configure fallback models
4. Generate `.env.local` or `ai-config.json`
5. Save to disk automatically

## What You Need

### 1. OpenRouter Account

- **Free signup**: https://openrouter.ai
- **Get API key**: Settings → API Keys → Copy your key (starts with `sk-or-`)

### 2. Environment

- **Local dev**: `.env.local` (generated by wizard)
- **Vercel/Production**: Environment variables in dashboard

## Step-by-Step Walkthrough

### Step 1: API Key Configuration

**Web UI:**
1. Visit https://openrouter.ai and create a free account
2. Copy your API key (format: `sk-or-...`)
3. Paste into the wizard
4. Click "Test Connection" to verify

**CLI:**
```
Enter your OpenRouter API Key: sk-or-...
Testing connection...
✓ Connection successful!
```

**Why test?** Ensures your key is valid before proceeding.

### Step 2: Primary Model Selection

The wizard offers 5 free models:

| Model | Provider | Ideal For | Max Tokens | Cost |
|-------|----------|-----------|------------|------|
| Claude 3.5 Haiku | Anthropic | Real-time tasks, fast inference | 8,000 | $0.8/1k |
| Mistral 7B | Mistral | Strong reasoning, balanced | 32,000 | $0.14/1k |
| LLaMA 2 7B | Meta | Open-source, general purpose | 4,096 | $0.1/1k |
| Phi 2 | Microsoft | Efficient reasoning | 4,096 | $0.2/1k |
| Airoboros L2 70B | Community | Advanced reasoning | 4,096 | $0.7/1k |

**Recommendation:**
- **For speed**: Claude 3.5 Haiku
- **For quality**: Mistral 7B or Airoboros
- **For cost**: LLaMA 2 7B

### Step 3: Fallback Chain Configuration

Set up 1-3 backup models for reliability.

**Example chain:**
1. Primary: Claude 3.5 Haiku (fast)
2. Fallback 1: Mistral 7B (better quality)
3. Fallback 2: LLaMA 2 7B (reliable budget option)

**How fallback works:**
```
Request → Try Primary Model
         ↓ (fails)
         → Try Fallback 1
         ↓ (fails)
         → Try Fallback 2
         ↓ (fails)
         → Error: All models exhausted
```

### Step 4: Preview Configuration

Review your settings in your preferred format:

**`.env.local` format** (recommended):
```env
OPENROUTER_API_KEY=sk-or-...
AI_PRIMARY_MODEL=anthropic/claude-3.5-haiku
AI_FALLBACK_MODELS=mistralai/mistral-7b-instruct,meta-llama/llama-2-7b-chat
AI_CONFIG_VERSION=1.0.0
```

**`ai-config.json` format**:
```json
{
  "ai": {
    "provider": "openrouter",
    "apiKey": "sk-or-...",
    "primaryModel": "anthropic/claude-3.5-haiku",
    "fallbackModels": [
      "mistralai/mistral-7b-instruct",
      "meta-llama/llama-2-7b-chat"
    ]
  }
}
```

### Step 5: Deploy

**Local Development:**
1. Download or copy configuration
2. Save to `.env.local` in project root
3. Run: `npm run dev`
4. Restart app to load variables

**Vercel (Production):**
1. Go to Vercel Dashboard
2. Project Settings → Environment Variables
3. Add each variable:
   - `OPENROUTER_API_KEY` = your API key
   - `AI_PRIMARY_MODEL` = model ID
   - `AI_FALLBACK_MODELS` = comma-separated IDs
4. Redeploy: `npm run deploy:prod`

**Other Platforms:**
Apply environment variables according to your platform's documentation.

## Using OpenRouter Models in Code

### Basic Usage

```typescript
import { getOpenRouterClient } from '@/lib/openrouter/client';

const client = getOpenRouterClient();

const response = await client.complete({
  model: client.getPrimaryModel(),
  messages: [
    { role: 'user', content: 'Hello!' }
  ],
  maxTokens: 1000,
});

console.log(response.choices[0].message.content);
```

### With Fallback Chain

Fallback is automatic — the client tries primary, then falls back to each model in order:

```typescript
// Automatically handles fallback
const response = await client.complete({
  messages: [{ role: 'user', content: 'Complex reasoning task' }],
  maxTokens: 2000,
});
// If primary fails → tries fallback 1, then fallback 2, etc.
```

### Track Usage

```typescript
import { getUsageTracker } from '@/lib/openrouter/usage-tracker';

const tracker = getUsageTracker();

// After each completion:
tracker.track(
  'anthropic/claude-3.5-haiku',
  promptTokens,
  completionTokens
);

// Get stats:
const stats = tracker.getStats();
console.log(`Total cost: $${stats.totalCost.toFixed(2)}`);
```

## Troubleshooting

### "Invalid API key"

**Solution:**
1. Check your API key starts with `sk-or-`
2. Go to https://openrouter.ai/account/api-keys
3. Create a new key if needed
4. Try again

### "Model not found"

**Solution:**
1. Verify model ID is correct from wizard
2. Check model availability at https://openrouter.ai/models
3. Some models may have access restrictions

### "Connection timeout"

**Solution:**
1. Check internet connection
2. Verify OpenRouter API is reachable: https://openrouter.ai/status
3. Increase timeout: `AI_TIMEOUT_MS=60000`
4. Try a different model

### Configuration not loading

**Local Development:**
```bash
# Clear cache
rm -rf .next node_modules/.cache

# Restart
npm run dev
```

**Vercel:**
1. Verify env vars in dashboard (Settings → Environment Variables)
2. Check if variable names match exactly
3. Redeploy: `npm run deploy:prod`

## Security Best Practices

### Local Development

**✓ DO:**
- Store API key in `.env.local`
- Add `.env.local` to `.gitignore` (already done)
- Use different keys for dev/prod
- Rotate keys regularly

**✗ DON'T:**
- Commit `.env.local` to git
- Log API keys to console
- Expose keys in client-side code
- Share keys via email/chat

### Production

**✓ DO:**
- Use Vercel Environment Variables
- Restrict key permissions to OpenRouter API only
- Monitor usage at https://openrouter.ai/account/activity
- Set up billing alerts if available

**✗ DON'T:**
- Use production key for local dev
- Share secrets with team directly
- Store in cookies or localStorage
- Log requests with keys

## Advanced Configuration

### Custom Model Costs

Update model costs in `/lib/openrouter/usage-tracker.ts`:

```typescript
const MODEL_COSTS: Record<string, number> = {
  'your-model-id': 0.001, // $0.001 per token
  // ...
};
```

### Custom Timeout

```env
# In .env.local or Vercel
AI_TIMEOUT_MS=60000  # 60 seconds
AI_MAX_RETRIES=5     # Try up to 5 times
```

### Load Balancing

Switch primary model dynamically:

```typescript
const client = getOpenRouterClient();
client.switchPrimaryModel('mistralai/mistral-7b-instruct');
```

### Config from File

Instead of environment variables:

```typescript
import { OpenRouterClient } from '@/lib/openrouter/client';
import aiConfig from '@/ai-config.json';

const client = OpenRouterClient.fromConfig(aiConfig);
```

## Monitoring and Analytics

### Usage Statistics

```typescript
const tracker = getUsageTracker();
const stats = tracker.getStats();

console.log(`
  Total Requests: ${stats.totalRequests}
  Total Tokens: ${stats.totalTokens}
  Total Cost: $${stats.totalCost.toFixed(2)}
  
  By Model:
  ${Object.entries(stats.byModel)
    .map(([model, data]) => 
      `${model}: ${data.requests} requests, ${data.tokens} tokens, $${data.cost.toFixed(2)}`
    )
    .join('\n  ')}
`);
```
