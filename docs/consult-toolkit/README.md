# DSG AI Governance Readiness Toolkit

Purpose: help consulting, security, compliance, and AI transformation teams use DSG as a practical toolkit for AI governance readiness assessment, implementation planning, and evidence demonstration.

Boundary: this toolkit supports readiness, implementation, and evidence workflows. It is not a certification report and does not claim ISO, NIST, or third-party audit certification.

## Toolkit contents

1. Client Intake Questionnaire
   - File: `docs/consult-toolkit/client-intake-questionnaire.md`
   - Purpose: collect buyer context, AI workflow scope, risk areas, and evidence needs.

2. AI Workflow Inventory Template
   - File: `docs/consult-toolkit/ai-workflow-inventory-template.md`
   - Purpose: list AI agents, automations, tools, datasets, actions, owners, and execution paths.

3. Risk Classification Checklist
   - File: `docs/consult-toolkit/risk-classification-checklist.md`
   - Purpose: classify AI actions by execution risk, data sensitivity, approval requirements, and audit need.

4. Client Report Template
   - File: `docs/consult-toolkit/client-report-template.md`
   - Purpose: produce a practical readiness report with findings, evidence gaps, DSG controls, and recommended next steps.

5. Implementation Playbook
   - File: `docs/consult-toolkit/implementation-playbook.md`
   - Purpose: guide a pilot from discovery to DSG plan-check, Monitor Mode, audit export, and deploy gate.

## Recommended consulting workflow

```text
Client intake
→ AI workflow inventory
→ Risk classification
→ Control mapping
→ DSG pilot path
→ Evidence export
→ Client readiness report
```

## DSG positioning for consultants

DSG helps consultants show how AI-proposed actions can be governed before execution using deterministic gates, invariant checks, approval/risk controls, and audit evidence export.

## Not claimed

- ISO certification
- NIST certification
- Independent audit certification
- Guaranteed compliance
- Universal replacement for workflow or automation platforms
