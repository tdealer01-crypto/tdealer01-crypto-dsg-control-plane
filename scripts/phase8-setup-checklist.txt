PHASE 1: CREDENTIAL GATHERING (10 min)
================================================================================

Gather these 11 credentials and save to a secure location (NOT in git)

SUPABASE CREDENTIALS
  [ ] NEXT_PUBLIC_SUPABASE_URL
      Source: Supabase Dashboard → Settings → API → Project URL
      Format: https://xxxxx.supabase.co

  [ ] NEXT_PUBLIC_SUPABASE_ANON_KEY
      Source: Supabase Dashboard → Settings → API → anon public
      Format: eyJ... (long JWT)

  [ ] SUPABASE_SERVICE_ROLE_KEY
      Source: Supabase Dashboard → Settings → API → service_role secret
      Format: eyJ... (long JWT, marked as secret)

STRIPE CREDENTIALS
  [ ] STRIPE_SECRET_KEY
      Source: Stripe Dashboard → Developers → API Keys → Secret key
      Format: sk_test_... or sk_live_...
      ⚠️  WARNING: This is a secret! Never commit to git.

  [ ] STRIPE_WEBHOOK_SECRET
      Source: Stripe Dashboard → Webhooks → Create endpoint
      Format: whsec_... (you'll create the webhook later)

  [ ] STRIPE_PRICE_PRO
      Source: Stripe Dashboard → Products → Pro pricing tier
      Format: price_... (long alphanumeric)

  [ ] STRIPE_PRICE_BUSINESS
      Source: Stripe Dashboard → Products → Business pricing tier
      Format: price_... (long alphanumeric)

UPSTASH REDIS CREDENTIALS
  [ ] UPSTASH_REDIS_REST_URL
      Source: Upstash Console → Database → REST API → URL
      Format: https://xxxxx.upstash.io

  [ ] UPSTASH_REDIS_REST_TOKEN
      Source: Upstash Console → Database → REST API → Token
      Format: long alphanumeric

RESEND CREDENTIALS
  [ ] RESEND_API_KEY
      Source: Resend Dashboard → API Keys → Copy any token
      Format: re_... (long alphanumeric)

GENERATED CREDENTIALS
  [ ] CRON_SECRET (generate with: openssl rand -hex 32)
      Format: 64 hex characters

APP IDENTITY
  [ ] NEXT_PUBLIC_APP_URL = https://tdealer01-crypto-dsg-control-plane.vercel.app
  [ ] APP_URL = https://tdealer01-crypto-dsg-control-plane.vercel.app
  [ ] DSG_CORE_MODE = internal

CREATE .env.local FILE
  [ ] Copy .env.example to .env.local
      Command: cp .env.example .env.local

  [ ] Fill in all 11 credentials in .env.local
      Command: nano .env.local (or use your editor)

  [ ] Verify syntax (no errors)
      Command: bash -n .env.local

  [ ] File is in .gitignore (should be already)
      Command: grep .env.local .gitignore

✓ PHASE 1 COMPLETE when all items above are checked

PHASE 2: DEPLOY TO VERCEL (20 min)
================================================================================

PUSH CODE TO MAIN
  [ ] Check current branch
      Command: git branch --show-current

  [ ] Ensure no uncommitted changes
      Command: git status

  [ ] Pull latest from main
      Command: git pull origin main

  [ ] Push to main (if you made changes)
      Command: git push origin main

MONITOR VERCEL DEPLOYMENT
  [ ] Go to Vercel Dashboard → Deployments
      URL: https://vercel.com/dashboard

  [ ] Deployment status shows "Ready" (wait 3-5 minutes)
      Look for: Green checkmark and "Ready" status

  [ ] Note the production URL
      Example: https://tdealer01-crypto-dsg-control-plane.vercel.app

SET ENVIRONMENT VARIABLES IN VERCEL (13 vars total)

  [ ] Go to Project Settings → Environment Variables
      URL: https://vercel.com/dashboard → Select project → Settings

  [ ] Set scope to "Production" for all variables

  SUPABASE (3 vars)
    [ ] NEXT_PUBLIC_SUPABASE_URL
    [ ] NEXT_PUBLIC_SUPABASE_ANON_KEY
    [ ] SUPABASE_SERVICE_ROLE_KEY

  STRIPE (4 vars)
    [ ] STRIPE_SECRET_KEY
    [ ] STRIPE_WEBHOOK_SECRET
    [ ] STRIPE_PRICE_PRO
    [ ] STRIPE_PRICE_BUSINESS

  UPSTASH (2 vars)
    [ ] UPSTASH_REDIS_REST_URL
    [ ] UPSTASH_REDIS_REST_TOKEN

  RESEND (1 var)
    [ ] RESEND_API_KEY

  SYSTEM (3 vars)
    [ ] CRON_SECRET
    [ ] NEXT_PUBLIC_APP_URL
    [ ] APP_URL

  DSG (1 var)
    [ ] DSG_CORE_MODE = "internal"

VERIFY ENV VARS ARE SET
  [ ] Run: vercel env ls production

TRIGGER REBUILD IN VERCEL
  [ ] Option A: Click "Redeploy" on latest deployment in Vercel UI

  [ ] Wait for new deployment to show "Ready" status (3-5 minutes)

  [ ] Note new deployment URL (should be same as production alias)

VERIFY HEALTH ENDPOINT
  [ ] Test health endpoint:
      Command: curl -fsSL https://tdealer01-crypto-dsg-control-plane.vercel.app/api/health | jq .

================================================================================
PHASE 3: VERIFY & ACTIVATE (15 min)
================================================================================

RUN SUPABASE MIGRATIONS
  [ ] Install Supabase CLI
      Command: npm install -g supabase

  [ ] Login to Supabase
      Command: supabase login

  [ ] Link to your Supabase project
      Command: supabase link --project-ref YOUR_PROJECT_REF
      (Find project ref in Supabase Dashboard → Settings → General)

  [ ] Apply migrations
      Command: supabase db push

  [ ] Verify migrations were applied
      Go to Supabase Dashboard → SQL Editor → Run:
      SELECT * FROM pg_migrations;
      Should show ~14 migrations in status 'Success'

CONFIGURE SUPABASE AUTH URLS (Manual in Dashboard)
  [ ] Go to Supabase Dashboard → Authentication → URL Configuration

  [ ] Set Site URL
      Value: https://tdealer01-crypto-dsg-control-plane.vercel.app

  [ ] Add Redirect URL
      Value: https://tdealer01-crypto-dsg-control-plane.vercel.app/auth/confirm

  [ ] Click Save

CONFIGURE STRIPE WEBHOOK (Manual in Dashboard)
  [ ] Go to Stripe Dashboard → Webhooks

  [ ] Click "Create new endpoint"
      Endpoint URL: https://tdealer01-crypto-dsg-control-plane.vercel.app/api/stripe/webhook

  [ ] Select events to listen for:
      - customer.subscription.created
      - customer.subscription.updated
      - customer.subscription.deleted
      - invoice.payment_action_required
      - meter.error_occurred

  [ ] Copy the "Signing secret" (starts with whsec_)

  [ ] Update STRIPE_WEBHOOK_SECRET in Vercel
      Command: vercel env add STRIPE_WEBHOOK_SECRET production
      (Paste the signing secret when prompted)

  [ ] Trigger another rebuild in Vercel
      Command: vercel --prod
      OR click Redeploy in Vercel UI

RUN AUTOMATED VERIFICATION SCRIPTS
  [ ] Go/No-Go gate check (comprehensive)
      Command: npm run go:no-go https://tdealer01-crypto-dsg-control-plane.vercel.app
      Expected: All checks pass with ✅

  [ ] Deployment verification (detailed report)
      Command: ./scripts/deployment-verification.sh https://tdealer01-crypto-dsg-control-plane.vercel.app
      Expected: Passed checks > 0, Critical failures = 0

  [ ] Smoke test suite
      Command: ./scripts/smoke-test-suite.sh https://tdealer01-crypto-dsg-control-plane.vercel.app
      Expected: TESTS_PASSED > 0, TESTS_FAILED = 0

TEST FIRST LOGIN (Manual)
  [ ] Open login page
      URL: https://tdealer01-crypto-dsg-control-plane.vercel.app/login

  [ ] Click "Start Free Trial" tab

  [ ] Enter test email (from your secure vault, NOT in git)

  [ ] Enter workspace name: "DSG Ops"

  [ ] Click "Start free trial"

  [ ] Check email for magic link from noreply@resend.dev

  [ ] Click the magic link in email

  [ ] Verify you're logged in to dashboard
      Should see: Agents, Policies, Usage, Executions, etc.

✓ PHASE 3 COMPLETE when:
  - All automated verification scripts pass
  - Supabase migrations applied successfully
  - Supabase auth URLs configured
  - Stripe webhook created and receiving events
  - First login succeeds with email magic link
